Week four lab

This lab will apply CSS from the week four reading and videos to your own web pages from assignment one.

HTML

  1. Create a new folder called weeks-1-3, and copy your existing code into that. This folder will have your CSS and images from weeks 1-3.

  2. Create a new folder called week-four and copy your existing code into that. This folder will have the CSS and images you're updating for week four.

  3. From now on, at the start of each week, make a new folder, copy the previous week's content into that, and do your work for the new week in the new folder.

  4. On your site root index.html:

    1. Clear out everything between <main> and </main>.

    2. Inside the <header> container (not the <head> container), add this:

      <nav>
      <ul>
      <li><a href="weeks-1-3/index.html">Weeks 1-3</></li>
      <li><a href="week-4/index.html">Week 4</a></li>
      </ul>
      </nav>

    3. Open that in the web browser and make sure the links work.

    4. From now on, for each week, add a link to the new week's folder and make sure the links work.

    5. Inside the <main> container on the main index.html page, a <section> container.

    6. Inside the section, add a paragraph that says this site will trace the evolution of what you're learning in the Web authoring class. That's mostly a placeholder, so feel free to expand on that to make it more interesting.

    7. Just before the closing <body> tag, add a <footer></footer> container.

    8. Inside the footer container, add a paragraph that contains the &copy; and your name. You can put more in here if you like.

    9. Copy the <footer> container from the main index page.

    10. In the week-4 folder, paste the footer container into both the index.html and the about-me.html pages.

    11. In the footer on the week-four/index.html page, hyperlink your name to the about-me page in that folder.

    12. Inside the <main> container on each page in the week-four folder, add at least one <section> container and move the HTML that's in the <main> into that <section> container.

      Old: <main> (lots of html) </main>

      New:<main><section>(lots of html)</section></main>

    13. If you have content on this page that should be in its own section, make another section and put it there. Repeat until the structure makes sense.

      <main>

      <section>

      (lots of html about my childhood)

      </section>

      <section>

      (lots of html about my job)

      </section>

      <section>

      (lots of html about wine)

      </section>

      </main>

    14. On the about-me.html page, create an <aside> container.

    15. Move the heading and <ul> list of your classes into the <aside> container.

    16. That's the minimum for HTML changes. Feel free to add more structural HTML elements, like an <article> or two, or additional <aside> or <section> containers.

CSS

From The box model and Backgrounds and borders:

Make these changes to your linked CSS:

  1. In the /styles/ folder off your main root, open style.css and modify the looks of your main index.html page. This page will look different from your week-by-week portfolio pages, so use different colors, fonts, or anything else that suits you.

    1. Dress up the header some. Color, padding, background-color, whatever.

    2. For the header nav ul and header nav ul li, turn it into a horizontal navigation bar by following the guidance under Horizontal Navigation Bar Examples on https://www.w3schools.com/css/css_navbar.asp.

  2. Create styles for the main page's footer: text, color, and padding, at a minimum.

  3. Create styles for section containers that will be on your main page: text, color, and padding, at a minimum.

  4. Now go into the style/style.css that's in the week-4 folder.

    1. Format the header styles, making them different from the ones you did on the main page.

    2. Format the footer styles, making them different from the ones you did on the main page.

    3. Format the section styles, making them different from the ones you did on the main page.

    4. Create styles for the rest of the containers you have in the week-4 folder, which will be an <aside> container at the very least. Give it a border, adjust the font-size so it's not the same as what's in a section, maybe give it a slight change in background-color or color, at a minimum.

  5. Add comments to your CSS to explain what the different parts do.

Images

From https://developer.mozilla.org/en-US/docs/Learn/HTML/Multimedia_and_embedding/Responsive_images:

  1. Go to https://picresize.com.

  2. If the image you used on your website is under 1200px wide, go find another one that is at least 1200px wide. Given the insane resolution of modern cellphone cameras, this should not be difficult.

  3. Create a copy of your image, at each of these sizes:

    • 320px wide
    • 480px wide
    • 800px wide
    • 1200px wide

  4. Save these images in the images folder in your new week-4 folder

  5. In your HTML in the week-4 folder, follow the example at https://developer.mozilla.org/en-US/docs/Learn/HTML/Multimedia_and_embedding/Responsive_images, under Resolution switching: Different sizes. Inside the <img> container, add srcset and sizes for the new images.

  6. Remember to add one for the 1200px image as well; that's not in the example.

The result should look something like this:

<figure>

<img srcset="images/cobalt-hair-320w.jpg 320w,

images/cobalt-hair-480w.jpg 480w,
images/cobalt-hair-800w.jpg 800w,
images/cobalt-hair-1200w.jpg 1200w"

sizes="(max-width: 480px) 280px,

(max-width: 800px) 440px,
(max-width: 1199px) 760px,
1200px"

src="images/cobalt-hair.jpg"
alt="I got a bright blue mohawk haircut for church choir">

<figcaption>I was trying to get out of having to participate in choir.</figcaption>

</figure>

Accessibility

  1. Install the Accessibility Insights extension for Chrome or Firefox.

  2. Check your pages.

    1. Fix what you can figure out

    2. Ask for help on Slack for anything you can't figure out.

    3. Upload your completed work to GitHub.

    Once uploaded, be sure to validate your pages.

    Once validated, do one more accessibility check, this time via the WAVE evaluation tool from Web AIM.
