HTML basics

  • Main page

HTML is like real life.

  • If you open a door, close it.
  • If you open a tag, close it.
  • You have a head and a body.
  • HTML has a head and a body.
  • Your head is not inside your body, and your body is not inside your head. Same thing with HTML. The head has to end before the body begins.
  • HTML is like a bodybuilder who took too many steroids. The body begins immediately after the head ends. It has no neck.

You stop doing things in reverse order from how you started them.

  1. Walk into the house
  2. Walk up the stairs
  3. Walk into the bathroom
  4. Pick up the toothbrush
  5. Brush your teeth

When done:

  1. Stop brushing your teeth
  2. Put down the toothbrush
  3. Walk out of the bathroom
  4. Walk down the stairs
  5. Walk out of the house

If it was HTML:

<in-house>

<stairs>

<bathroom>

<toothbrush>

<brush-teeth>

</brush-teeth>

</toothbrush>

</bathroom>

</stairs>

</in-house>

HTML stops doing things in reverse order from how you started them.

<html>

<head>

<title>Nesting</title>

</head>

<body>

<h1>Nesting: closing tags reverse the <em>order</em> from opening tags</h1>

<p>Always <strong>close what you open, or <em>finish the last thing you were doing</em></strong> before you get distracted.</p>

</body>

</html>

Since pages are usually part of a site, there's likely going to be something that contains stuff like the site title, the logo, and the main navigation. That something will be the <header> container.

It's not the same as the <head>.

It's in the <body>, but not inside the <main> container.

<html lang="en">

<head>

<title>Example HTML page</title>

<meta charset="UTF-8">

</head>

<body>

<header>

<h1>Web authoring</h1>

<h2>A really excellent class</h2>

</header>

I happen to know that I want:

  • distinct styling for the 2nd paragraph,
  • something else for the 4th paragraph, and
  • something completely different for the second sentence in the third paragraph.

I need to do something with the HTML so I can add styles later.

Since the paragraphs are already contained in an HTML tag, I'm going to give them a class name so I can refer to it later.

Since the second sentence in the third paragraph is not in a container, I'm going to put it in one that I can refer to later. Since the sentence doesn't start and stop on its own line, I'm going to use a <span> container. That's what you use for inline items. Otherwise I'd use a <div>.

<html lang="en">

<head>

<title>Example HTML page</title>

<meta charset="UTF-8">

</head>

<body>

<header>

<h1>Web authoring</h1>

<h2>A really excellent class</h2>

</header>

<main>

<h1>Every page needs a top-level h1 heading</h1>

<p>A heading should have some content after it.</p>

<p class="more-content">Maybe some more content than that.</p>

<p>How much is enough? <span class="who-knows">Does anyone really know?</span></p>

<p class="beats-me">I can't say. I don't know everyone.</p>

</main>

</body>

</html>

Are we done yet? Here's some stuff for later:

Sitewide navigation will go inside that <header> container.

Inside the <main> container, you can have multiple <section> containers, and other sectional elements as well. You can read more about structural HTML (aka sectional elements) at https://developer.mozilla.org/en-US/docs/Web/Guide/HTML/Using_HTML_sections_and_outlines.

Meanwhile, let's be done with HTML for a while and switch to style. So, the last thing we do to the HTML is to tell it where to go for style.

Let's also put a couple comments in the code to help document what we've done.

<html lang="en">

<head>

<title>Example HTML page</title>

<meta charset="UTF-8">

<link rel="stylesheet" href="styles/style.css">

<!-- now it's time to go deal with CSS -->

<!-- that's a comment that won't display on the page -->

<!-- comments start with a < ! - - and they end with two dashes and a > -->

</head>

<body>

<header>

<h1>Web authoring</h1>

<h2>A really excellent class</h2>

</header>

<main>

<p class="more-content">Maybe some more content than that.</p>

<p>How much is enough? <span class="who-knows">Does anyone really know?</span></p>

CSS for the win!

/* CSS page */

/* in a folder called styles, and named style.css, since that's how I wrote the HTML */

/* Comments in CSS start with a slash and an asterisk, and end with an asterisk and a slash */

/* We apply styles to HMTL tags by just putting the tag names here, no < or > involved */

body { color: darkslategray } /*font color */

header { background-color: navy; color: white }

.more-content { font-style: italic; color: navy } /* A class in HTML starts with a dot in CSS */

.who-knows { font-weight: bold; font-color: red }

.beats-me { color: gray }

Web development process

It differs depending on what I'm trying to do and what tools I am using, but at its simplest I have just two tools: something to edit code, and a web browser.

  1. I open both a code editor and a web browser: for example Visual Studio Code and Google Chrome.

    If the screen is big enough, I open them so I can see both at once (see attached screen shot).

  2. I will type out or paste in some code that has the basic structure I need, like this:

    <html>
      <head>
       <link rel="stylesheet" type="text/css" href="styles/style.css" />
        <title>Project Unicorn</title>
      </head>
      <body>
        <heading>
          <h1>Unicorns and Zombies</h1>
        </heading>
        <main>
          <section>
            <p>
              Researchers disagree on which is more awesome: unicorns or zombies.
            </p>
          </section>
        </main>
      </body>
    </html>

  3. I'll save that as a file whose filename ends in .html

  4. I will put the latest version of my working files into GitHub. This includes folders, CSS files, images, etc.

  5. I will open my HTML page (the copy on my computer, not the copy on github.io--so I can edit and review quickly) in the web browser to see if it looks right.

  6. If it doesn't look right:

    1. I will look at my code to see if I can find a mistake.

      • If I change my code, I will save it in the editor, and click refresh in the browser to see if that fixed it.

    2. If I don't see anything obvious, I will paste the code into the validator at https://validator.w3.org and see if it tells me anything helpful.

      • If I change my code, I will save it in the editor, and click refresh in the browser to see if that fixed it.

    3. If I still can't find what's wrong, I will send the GitHub link to someone else to see if they can see the issue.

  7. If it does look right, I will look at the page on GitHub.io to make sure it looks right there as well.

  8. If it doesn't look right, I will loop back to step 6 above.

  9. If it does look right, I will add more to the code: perhaps a new section, another heading or paragraph, an image, or some changes to the style.css page.

  10. In the editor, I will click "save" to save the code.

  11. I will loop back to step 4 above.
