Run a web server on your own computer

Once your folder structure starts getting complicated and we start using includes, we have a new problem that will affect people who are doing fancy stuff like reusing the include link from multiple folders. A link that works from one folder might not work from a subfolder.

If that's not a problem you are having, then that's fine. It's mostly going to be a problem for people who are going beyond the minimums described in the lab instructions.

If it's a problem you are having, the solution is start every hyperlink with a / like this: 

<a href="/week-04/index.html">Week 4</a>

That slash says, "from the site root, go to..." Now the navigation should work no matter what folder it's in, and you should only need the one file.

Important! Important! Important! That slash is the root of your website.

  • If your website is https://alboss-cascadia.github.io/ then /week-04/index.html will open https://alboss-cascadia.github.io/week-04/index.html.

  • However, if your website is https://alboss-cascadia.github.io/bit112-fall-2019/ then you need to use /bit112-fall-2019/week-04/index.html to get from the site root to the right place.

Be sure to put that same / here:

<script src="/scripts/includes.js"></script>

(using the same important important important comment about site root as above).

If you want it to use the main CSS and not something in a week folder, change it here: 

<link rel="stylesheet" type="text/css" href="/styles/style.css" />

(still using the same important important important comment about site root as above).

Great, Al, you broke my pages

When you start a link with a slash, that is shorthand for From the server root, go to... That works fine when the files are on github.io but you won't be able to browse them or see the CSS on your computer.

Some people are fine putting everything into GitHub to see how it looks. If that's you, great!

Other people want to see how it looks without having to upload to GitHub all the time. The solution is to install a Web server on your computer. There are lots of complicated ways to do this, but the simplest way is to use a free product that does all the complicated stuff for you. There are plenty of these. I'm suggesting MAMP because it's free and it works on both Windows and Mac OS.

Note that we are not turning your computer into a web server on the Internet.

  1. Go to https://www.mamp.info/en/" and pick the free download.

    They'll want to sell you the paid product but the free one is fine.

    (There is a $1.99 version in the Mac App store, if you want to go that route. That'll work equally well for Mac users, and might be less hassle.)

  2. Once the installation is complete, you’ll find two folders in your Applications (Mac) or Programs (Windows) folder — one for MAMP, and another that is a 14-day free trial for MAMP Pro. Open the folder named MAMP and launch the MAMP app.

  3. When you install MAMP, it sets the default Document Root folder on your computer. You can change this by going to the Web Server tab and clicking Select. Change the default Document Root for your MAMP website files by clicking the Select button and browsing to wherever you're storing your files for BIT 112.

  4. Click the “Start Servers” button in MAMP. That should start the web server and load your index.html page, and all the includes and links should work the same as they will on github.io.

  5. Change code, save it, reload the page in the browser, and things should all work.

  6. Holy $#&%, you just installed a working web server! Tell your boss you deserve a raise, update your resume and LinkedIn page, and show all your teachers the genius certificate you will automatically get sent from Elon Musk.

    Genius certificate not guaranteed. Or even likely. But you do deserve it.
