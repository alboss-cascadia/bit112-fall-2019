Week six lab

This lab will apply CSS from the week six reading and videos to your own web pages from assignment one.

HTML

assign style guide

  1. Create a new folder called week-6, and copy everything from week-5 into that. This new folder should now have your completed HTML, CSS and images from week 5, so you can update it for week 6.

  2. On your site root index.html:

    1. Just before the closing </body> on your index.html page, paste in these lines:

      <!-- Enable includes -->
      <script src="scripts/includes.js"><script>
      <script>
      includeHTML();
      </script>

    2. Go to the <nav> container.

      1. Just after the closing </nav> tag, paste this:

        <div w3-include-html="nav.html">
        </div>

      2. Cut the contents of the entire <nav> container.

  3. Create a new file in your main folder, named nav.html, and paste the <nav> container contents into that.

  4. Inside the <nav> container, add a link to the week-6 folder:

    <nav id="bit112-nav">
    <ul>
    <li><a href="weeks-1-3/index.html">Weeks 1-3</></li>
    <li><a href="week-4/index.html">Week 4</a></li>
    <li><a href="week-5/index.html">Week 5</a></li>
    <li><a href="week-6/index.html">Week 6</a></li>
    </ul>
    </nav>

  5. Save both nav.html and index.html.

  6. Create a new folder named scripts, off the site root.

  7. In the scripts folder, create a new file. Name it includes.js

  8. Copy the contents of the the file at https://github.com/alboss-cascadia/bit112-fall-2019/blob/master/scripts/includes.js and paste it into the new includes.js file you created.

  9. Sync your work to GitHub. It should now include

    • scripts/includes.js
    • nav.html
    • index.html with the new changes you made
    • the week-6 folder and all the files therein

  10. Wait a few minutes for the changes to sync to your site on GitHub.io and then load the main index.html page in your web browser. The navigation should still be visible, including the new week 6 link you just added.

    This type of include will not show up on your computer, so you do have to view it on the server. It's worth it since it'll save you typing the same navigation on multiple pages.

  11. Go to https://github.com/alboss-cascadia/bit112-fall-2019/blob/master/example.html

    1. Save the contents onto your site as template.html.

    2. Edit the title, h1, and footer so they're yours and not mine.

    3. Base any new files you create off this template, and they will automatically include the nav.html. Better still, any changes you make to your navigation will automatically show up in all files the used the template.

  12. That's the minimum for HTML changes. Feel free to add more.

CSS

Make some changes to your main style/style.css.

  1. Create positioning declarations (padding, margin, display, float, position, width, height, etc.) for the Header, Nav, Main, Footer sections on your main index.html page.

  2. Create a rule for the list items in the navigation, using the nav ID you created earlier. You've already set up navigation to be horizontal, so this is just moving that CSS to the rule for the ID, and maybe adjusting things a little.

    1. Set the display for the list items to be inline-block

    2. Use padding and maybe a border-right or border-left to make the navigation look okay.

    3. Set a uniform width for the list items in your navigation.

Make these changes to your linked style/style.css that's in the week-5 folder:

  1. Create a rule for the figure container, using the section ID you created earlier. Float the figure container to the right or left. Add some padding so the text doesn't run right up against the image.

    section#porpoise figure {
    float: left;
    padding-right: 2em;
    }

    You might find that the responsive image is too big for things to float next to it. You can manage that by setting a percentage width, for example:

    section#porpoise figure img {
    width: 60%;
    }

    It will still use the responsive image, but display it at 60% instead of 100% of its width.

  2. Create a rule for the section following the one with the image, using clear:left so that this next section ignores the float, and moves back into the normal page layout flow.

    section#after-image {
    clear: left;
    }

  3. Create at least ten positioning declarations (padding, margin, display, float, position, width, height, etc.) for the Header, Nav, Main, Footer sections.

  4. Use relative positioning to offset the .intro class you created earlier, and change its size and color or something. Here's my example.

    .intro {

    position: relative
    /* relative positioning means move it this far from where it would normally be */
    top: 20px;
    left: 20px;
    font-size: 1.5em;
    color: #333333;

    }

  5. Using https://developer.mozilla.org/en-US/docs/Learn/CSS/CSS_layout/Multiple-column_Layout as a guide, set the bit-columns class to display in three columns.

  6. Using https://developer.mozilla.org/en-US/docs/Learn/CSS/CSS_layout/Media_queries as a guide, change the bit-columns class

    1. to two columns when the min-width is 321px and the max-width is 600px.

    2. to one column when the width is less than 321px
      hint: unset is your friend. https://developer.mozilla.org/en-US/docs/Web/CSS/unset 

  7. Add comments to your CSS to explain what the different parts do.
