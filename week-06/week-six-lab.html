Week six lab

Things are getting serious. Flexbox, CSS Grid, columns, responsive design, and Javascript includes.

HTML

  1. Create a new folder called week-6, and copy everything from week-5 into that. This new folder should now have your completed HTML, CSS and images from week 5, so you can update it for week 6.

  2. On your site root index.html:

    1. Just before the closing </body> on your index.html page, paste in these lines:

      <!-- Enable includes -->
      <script src="scripts/includes.js"><script>
      <script>
      includeHTML();
      </script>

    2. Go to the <nav> container.

      1. Just after the closing </nav> tag, paste this:

        <div w3-include-html="nav.html">
        </div>

      2. Cut the contents of the entire <nav> container.

  3. Create a new file in your main folder, named nav.html, and paste the <nav> container contents into that.

  4. Inside the <nav> container, add a link to the week-6 folder:

    <nav id="bit112-nav">
    <ul>
    <li><a href="weeks-1-3/index.html">Weeks 1-3</></li>
    <li><a href="week-4/index.html">Week 4</a></li>
    <li><a href="week-5/index.html">Week 5</a></li>
    <li><a href="week-6/index.html">Week 6</a></li>
    </ul>
    </nav>

  5. Save both nav.html and index.html.

  6. Create a new folder named scripts, off the site root.

  7. In the scripts folder, create a new file. Name it includes.js

  8. Copy the contents of the the file at https://github.com/alboss-cascadia/bit112-fall-2019/blob/master/scripts/includes.js and paste it into the new includes.js file you created.

  9. Sync your work to GitHub. It should now include

    • scripts/includes.js
    • nav.html
    • index.html with the new changes you made
    • the week-6 folder and all the files therein

  10. Wait a few minutes for the changes to sync to your site on GitHub.io and then load the main index.html page in your web browser. The navigation should still be visible, including the new week 6 link you just added.

    This type of include will not show up on your computer, so you do have to view it on the server. It's worth it since it'll save you typing the same navigation on multiple pages.

  11. Go to https://github.com/alboss-cascadia/bit112-fall-2019/blob/master/example.html

    1. Save the contents onto your site as template.html.

    2. Edit the title, h1, and footer so they're yours and not mine.

    3. Base any new files you create off this template, and they will automatically include the nav.html. Better still, any changes you make to your navigation will automatically show up in all files the used the template.

  12. That's the minimum for HTML changes. Feel free to add more.

Flexbox

  1. Go to https://github.com/alboss-cascadia/bit112-fall-2019/blob/master/week-06/flexbox-01.html and copy the code into a new page.

  2. Save this page in your week-6 folder. It's for a Flexbox demo.

  3. Add a link to this page from your week-6/index.html page.

  4. Open the page in the web browser.

  5. Scroll to the bottom and see what's going on

  6. In the style container on this page, change the display for section from display: block to display: flex and save the page.

  7. Refresh the page in your browser, scroll down, and be amazed how easy that was.

    What just happened?

    We wanted the articles to be laid out as flexible boxes. To make that happen, you set the parent element (section, in this case) to be a flex container.

    But, uh-oh, that section with the 12 articles doesn't fit. Now you have to scroll sideways to see them. That's a problem. And notice that it's gotten bigger than the section--the purple dotted line is the end of the section.

  8. Fix the third section so the articles fit nicely.

    1. In the CSS for #section-3, add a rule that sets flex-wrap: wrap;

    2. In the CSS for #section-3 article, add a declaration that sets flex: 250px;, and save the page

    3. Refresh the page in your browser, scroll down, and be amazed how easy that was.

      What just happened?

      We got rows â€” flexbox children (articles in this case) go into a row until the row is full, and any overflow is moved down to the next line. The flex: 250px on the articles means each will be at least 250px wide. On the last row, they're wider so that the entire row is still filled.

  9. Change the column sizes in the middle section

    1. In the CSS for #section-2 article, add a declaration that sets flex: 1;

    2. Make a new rule below the one for #section-2 article. We want to make the third article be twice as wide as the others, no matter what size.

      #section-2 article:nth-of-type(3) {
      flex: 2;
      }

    3. Save the code, refresh the page in your browser, scroll down, and be amazed how easy that was.

      What just happened?

      The flex: 1 basically just says give everything an equal proportional space. The number is pretty arbitrary.

      The article:nth-of-type(3) pseudoclass means find the 3rd article, and flex: 2 means make it twice as wide as the others.

      You can use px values or ems instead, if you prefer. Proportions just means "I don't care that much about the exact side and I'm willing to let the computer do the math."

CSS-Grid

  1. Go to https://github.com/alboss-cascadia/bit112-fall-2019/blob/master/week-06/grid-01.html and copy the code into a new page.

  2. Save this page in your week-6 folder. It's for a CSS-Grid demo.

  3. Add a link to this page from your week-6/index.html page.

  4. Open the page in the web browser.

  5. Scroll to the bottom and see what's going on

  6. In the style container on this page, change the display for .container from display: block to display: grid and save the page.

  7. Refresh the page in your browser, scroll down, and be amazed how easy that was.

    What just happened?

    Oh. Nothing. Declaring display: grid gives you a one column grid, so your items will look the same as they did in normal flow.

  8. In the .container rule, below the display: grid, add a second declaration:

    grid-template-columns: 200px 200px 200px; and save the page.

  9. Refresh the page in your browser, scroll down, and be amazed how easy that was.

    What just happened?

    With luck, the items have rearranged themselves one into each cell of the created grid.

  10. Fine, but pixels are boring.

    Change grid-template-columns: 200px 200px 200px; to grid-template-columns: 2fr 1fr 1fr; and save the page.

  11. Refresh the page in your browser, scroll down, and be amazed how easy that was.

    What just happened?

    The fr> unit distributes space in proportion. The first track now gets 2fr of the available space and the other two tracks get 1fr, making the first track larger. You can mix fr units and fixed length tracks.

  12. Space me out.

    Add this to the .container rule, below the grid-template-columns declaration:

    grid-gap: 20px;
    gap: 20px;

  13. Save the code, refresh the page in your browser, scroll down, and be amazed how easy that was.

    What just happened?

    We got space between the grid items.

    They're the same thing. The spec is changing from grid-gap to gap, so for now we use both so all the browsers will cooperate.

There is a ton more to CSS-Grid, but this is an intro class and I don't want to leave everyone with nothing left to learn. The reading at https://developer.mozilla.org/en-US/docs/Learn/CSS/CSS_layout/Grids goes into a lot more detail if you want to apply more of this to your project.
