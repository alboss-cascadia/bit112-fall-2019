GitHub and the Visual Studio Code editor

It's time we used these tools a little more efficiently, by connecting the code editor directly to GitHub. If you're using an editor that isn't Visual Studio Code, the instructions will differ, but the concepts will be the same.

Setup (one-time)

  1. Go to https://git-scm.com, download and install Git on your computer.

  2. Upload all your working files, regardless of what awful condition they might be in, to your GitHub repository.

  3. Go to your repo in GitHub.com. Copy the URL from the browser address bar. It will be structured like this:

    https://github.com/alboss-cascadia/bit112-fall-2019

    It starts with https://github.com/
    Then it has your github login
    then it has the name of your repo

  4. Copy that URL. (Yours, not mine.)

  5. Go to Visual Studio Code.

  6. Just to make sure it's a clean start, go to File, New Window.

  7. Click View, Command Palette.

  8. In the field that pops up, type in git and click on Git: Clone

  9. Paste in the URL.

  10. It will ask you where you want to put the repository. Don't put it inside the folders you have been working on for this class, or you will wind up with a copy inside the original.

  11. Click File, Save Workspace As, give it a name, and save it.

From now on

  1. On the sidebar in Visual Studio Code, you should notice the third icon down (three circles connected by 2 lines). That's the GitHub icon. It probably has a number next to it, to show you have changes that aren't in Git.

  2. Click that icon. It will show all the files you've changed.

  3. There's a little field at the top that says Message. This is where you type in what you have changed. For this one you can just type Setup since that's what you're doing. Normally you'd by typing something like New navigation formatting or Fixed typo on title tag.

  4. After you put in the message, click the checkmark just above the message box. If it says There are no staged changes to commit. Would you like to automatically stage all your changes and commit them directly? just click Yes.

  5. In VS Code, in the bottom left of the window, there should be the little git icon (three circles connected by 2 lines) and the word Master, and next to that there's an icon. It's probably two arrows forming a circle, although it could be a cloud. Click that.

    If it asks you for your GitHub username and password, enter them in the box when asked.

    This will add your changes to GitHub.com and update your github.io website.
