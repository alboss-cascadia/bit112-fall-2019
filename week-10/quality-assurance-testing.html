Quality assurance testing

What to do

For your work, the testing process will have two parts: HTML/CSS Validation and a Quality Assurance (QA) review. Complete the validation process on your own site, and complete the QA review of other students' sites. (They'll be assigned to you automatically as people turn them in for review this week.)

HTML validation errors related to google fonts can safely be ignored.

HTML/CSS validation uses a computerized checklist to carefully test the structure of a webpage and report errors. The official W3C validator is online at http://validator.w3.org/

The Web Developer browser extension/toolbar also has a W3 validation menu option built into it, and most of the browser developer tools and consoles flag validation errors.

Validate all the pages in your site, and make any changes required to make your pages compliant with the HTML/CSS standards. Note: you should have been doing this all along, so if you have been following instructions all along, then this isn't going to be a very big task.

A Quality Assurance (QA) review is another important part of testing a site. The process evaluates basic functionality, spelling, grammar, browser compatability, and clarity, along with making sure the overall presentation fits the tone of the site. QA Testing also makes a list of known errors and issues needing correction. We will be using GitHub's Issues function to list those for the sites you review.

If you want to take notes on paper first, check the checklist template and the checklist example.

How to do it

  1. Log into GitHub.
  2. On GitHub, navigate to the main page of the repository you're reviewing.
  3. Under the repository name, click Issues.
  4. Click New issue.
    • If there are multiple issue type templates available in the repository, click Get started next to the type of issue you'd like to open. That only happens if the repo owner has created templates for different issue types. Most of the time in this class, that's not going to happen, and GitHub will just open a new issue automatically for you.
  5. Type a title and description for your issue. In the description, include:
    1. Page title
    2. Filename
    3. URL
    4. Type of issue:
      1. Page Functionality Problems
        1. links
        2. rollovers
        3. scripts
      2. Content Problems
        1. spelling
        2. grammar
        3. clarity
      3. Design Problems
        1. fonts
        2. colors
        3. layout
    5. Description of issue
  6. When you're finished, click Submit new issue.
  7. Repeat until done.
